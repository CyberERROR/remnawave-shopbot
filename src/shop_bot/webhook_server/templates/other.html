Низкий % (0-10):
Система старается не использовать подкачку, пока RAM не заполнится почти полностью. Рекомендуется для баз данных (MySQL, Postgres) для лучшей производительности.

Высокий % (60-100):
Система проактивно сбрасывает неактивные данные в SWAP, освобождая RAM для файлового кэша. Стандартное значение: 60.
